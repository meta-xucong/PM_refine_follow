# Polymarket 账户 V2.1 重评分综合报告

## 一、说明

本次重评分基于 `polymarket_account_screening_standard_V2_1.md`。
和上一版 V2 相比，这次**只把“多子市场风险”的处理升级为 V2.1 的第一性原则**：

- 不再因为历史上买过多个互斥/递进子市场就直接重罚；
- 优先看是否**实质性同时持有多个相关腿**；
- `exclusive_sequential_switch` 与 `nested_sequential_roll` 只作轻中度扣分；
- `exclusive_concurrent_multi_leg` 与 `nested_concurrent_ladder` 仍作为高风险结构行为。

因此，这份 V2.1 重评分是一次**针对新规则的定向重算**：
其他维度（双边买入、同 token 快进快出、可利用话题密度、集中度、持仓时长）的口径保持与 V2 一致；
核心新增的是：对 raw trades 重新识别 `exclusive / nested` 事件里，究竟是**并存多腿**，还是**清仓后换仓 / 顺序 roll**。

## 三、整体结论

- **主池**：Optimus.
- **补充池**：aff3, Moond, kekkone
- **观察池**：leCommissaire
- **禁用**：Toncar16, KBO30, Zippy, Nostdam, 0x5095…93ee, intrepid

本次 V2.1 下，真正受益于“顺序换仓 / 顺序 roll 予以放宽”的，主要是：
- **Toncar16**：较 V2 上调 **2.3** 分
- **Nostdam**：较 V2 上调 **1.5** 分
- **Moond**：较 V2 上调 **1.2** 分
- **leCommissaire**：较 V2 上调 **0.7** 分
- **Optimus.**：较 V2 上调 **0.5** 分

但也要注意：V2.1 并不是“放宽所有 multi-market 行为”，它只是把风险源头从“历史上买过多个子市场”改成“是否实质性同时持有多个相关腿”。
所以像 `KBO30`、`0x5095…93ee`、`intrepid` 这种**并存互斥腿本身就很重**的账号，分数不会明显改善。
