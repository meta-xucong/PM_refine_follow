# Polymarket 账户评分含义说明（V2.2 锚点版）

## 一、评分构成（五个维度）

V2.2 评分体系由以下五个子分数加权组合而成，再经集中度惩罚、低频上限和锚点校正，得到最终分数。

| 维度 | 权重范围 | 含义 |
|:---|:---|:---|
| **Copyability Score** | 0~35 | 账户行为的可模仿程度。包括双边买入比例（是否同时押注对立面）、不可复制快交易比例、代币级快进快出行为等。越干净，分数越高。 |
| **Deployability Score** | 0~20 | 资金的可部署性/事件覆盖广度。核心指标是 `deployable_event_equivalent`（可利用事件等价值）和 `deployable_event_density`（密度）。交易的事件越多、分布越均匀，分数越高。 |
| **Multi-Market Structure Score** | 0~20 | 多市场结构健康度。衡量同一事件下是否存在互斥腿（exclusive concurrent leg）或嵌套期限梯（nested concurrent ladder）的并发持仓风险。结构越干净，分数越高。 |
| **PnL Curve Stability Score** | -15~+15（实现中映射为 0~15 贡献） | 收益曲线稳定性。基于官方 API 的已实现盈亏时间线，评估全时段/30天/7天三个窗口的趋势平滑度、最大回撤、波动性。长期稳健上行加分，下行或剧烈波动扣分。 |
| **Risk Penalty Adjustment** | -20~0 | 风险惩罚项。若出现极端集中度、缺失关键数据、或不可复制交易占比过高等保守假设，会扣除一定分数。 |

## 二、从子分数到最终分数的计算流程

1. **Raw Score（原始分）**
   ```
   raw = copyability + deployability + structure + pnl_curve + risk_penalty - concentration_penalty
   raw = clamp(raw, 0, 100)
   ```
   - **Concentration Penalty（集中度惩罚）**：若头部事件买入占比过高（如 top1 > 50% 且事件数 < 5），额外扣 5 分；top3 > 80% 且事件数 < 8，再扣 5 分。
   - **Low-Frequency Cap（低频上限）**：若交易的事件数太少或密度太低，分数会被硬性封顶（55 / 62 / 70），防止低频账户因运气获得虚高分。

2. **Anchored Score（锚点校正分）**
   ```
   anchored = clamp(raw + score_offset, 0, 100)
   ```
   - 系统选定一个**锚点账号**（`0x39d0...4fe8`），将其原始分强制对齐到 **60 分及格线**。
   - 本次批次的 `score_offset = +52.05`，意味着所有账户的原始分统一上浮 52.05 分后再截断到 0~100。
   - **Anchored Score 的意义**：它不是绝对的'满分 100'竞赛排名，而是**相对于锚点及格线的相对表现**。60 分表示'刚好达到锚点账号的及格水准'，高于 60 表示比锚点更好，低于 60 表示还不如锚点。

3. **Hard Exclusion（硬排除）**
   - 若账户触发任何一项硬排除条件（如双边买入比例 > 40%、互斥并发腿比例 > 35%、不可复制快买比例 > 20% 等），即使 anchored 分超过阈值，最终决策也会被强制压回 `not_recommended`。**分数高不代表可以跟，结构风险才是守门员。**

## 三、最终分数段含义（基于 anchored_score）

| 分数段 | 决策标签 | 中文含义 | 适用建议 |
|:---|:---|:---|:---|
| **90~100** | `relative_copyable` | 极为优秀 | 行为极其干净、结构风险极低、可复制性极强。可相对放心地作为跟单标的，但仍建议设置止损和仓位上限。 |
| **75~89** | `relative_copyable` | 良好可跟 | 整体表现优于锚点及格线，具备较好的跟单价值。若未触发硬排除，可作为核心观察池。 |
| **60~74** | `selective_copying_only` | 及格但需筛选 | 刚好达到或略优于锚点水平。存在某些结构性瑕疵（如事件密度偏低、近期 PnL 偏弱、或有轻微并发风险）。只建议在严格事件白名单/黑名单约束下**选择性**跟随，不宜全盘复制。 |
| **40~59** | `not_recommended` | 不建议跟 | 明显低于锚点及格线，或虽然分数因锚点偏移被拉高，但触发了硬排除条件。跟单风险收益比不佳。 |
| **0~39** | `not_recommended` | 不值得跟 | 行为模式高度不可复制，或存在严重结构风险（高频双边下注、极端集中度、大量快交易）。应直接排除。 |

## 四、Raw Score vs Anchored Score 的读取差异

- **Raw Score**：反映的是该账户在 V2.2 原始评分标准下的'绝对行为质量'。如果没有锚点系统，旧版最终得分就是 Raw Score。
- **Anchored Score**：反映的是该账户相对于一个被冻结的'60 分基准账号'的相对优劣。它让评分更具可比性和稳定性，避免市场整体波动导致评分基准漂移。
- **Delta vs Anchor 60**（`anchored - 60`）：直接告诉你该账户比锚点及格线强多少（正数）或弱多少（负数）。

---
*说明：本版评分采用锚点校正，锚点账号原始分 7.95 被冻结映射为 60 分，所有账户统一享受 +52.05 的偏移。*
